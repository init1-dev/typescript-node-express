Hotel Miranda node + express API

In our Node application, there are four main routes: Bookings, Rooms, Employees, and Messages.

Each route provides access to specific data and functionalities related to its corresponding entity and is designed to handle requests for its respective entity, providing comprehensive access to data stored within the application.

The GET ALL endpoint fetches all elements, while the GET ONE endpoint retrieves a specific element identified by its unique identifier. In case a request is made for a non-existent element, the INVALID endpoint gracefully handles the error by returning an appropriate message.

These routes and endpoints facilitate seamless interaction with the application, allowing users to retrieve, view, and manage data efficiently.

  • Rooms

    GET ALL | GET ONE | INVALID

  • Bookings

    GET ALL | GET ONE | INVALID

  • Employees

    GET ALL | GET ONE | INVALID

  • Messages

    GET ALL | GET ONE | INVALID
