In our Node application, there are four main routes: Bookings, Rooms, Employees, and Messages.

Each route provides access to specific data and functionalities related to its corresponding entity and is designed to handle requests for its respective entity, providing comprehensive access to data stored within the application.

The GET ALL endpoint fetches all elements, while the GET ONE endpoint retrieves a specific element identified by its unique identifier. In case a request is made for a non-existent element, the INVALID endpoint gracefully handles the error by returning an appropriate message.

These routes and endpoints facilitate seamless interaction with the application, allowing users to retrieve, view, and manage data efficiently.

Errors

This section provides a comprehensive list of common error codes that the API may return along with brief descriptions of each. Error codes play a crucial role in communicating issues between the API server and clients, aiding in troubleshooting and resolving problems efficiently. Understanding these error codes and their meanings can greatly enhance the development and debugging process when interacting with the API.

Code Meaning
400 Bad Request: The server cannot process the request due to a client error (e.g., malformed request syntax, invalid parameters).
401 Unauthorized: The request lacks valid authentication credentials or the authentication credentials are invalid.
403 Forbidden: The server understood the request but refuses to authorize it. Usually returned when authentication is provided, but the user doesn't have sufficient permissions.
404 Not Found: The requested resource could not be found on the server.
405 Method Not Allowed: The HTTP method used in the request is not supported for the requested resource.
409 Conflict: Indicates that the request could not be completed due to a conflict with the current state of the target resource.
422 Unprocessable Entity: The server understands the content type of the request entity, but it cannot process the contained instructions.
500 Internal Server Error: A generic error message indicating that something went wrong on the server's end.
502 Bad Gateway: The server was acting as a gateway or proxy and received an invalid response from the upstream server.
503 Service Unavailable: The server is currently unable to handle the request due to temporary overloading or maintenance of the server.
504 Gateway Timeout: The server was acting as a gateway or proxy and did not receive a timely response from the upstream server.
